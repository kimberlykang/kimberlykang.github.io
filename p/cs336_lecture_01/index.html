  1. Tokenization

Tokenization

  • Tokenizer: A class that performs encode and decode operations.
  • Vocabulary size: The number of distinct token types.

Tokenization Examples

You can try it yourself at tiktokenizer.

  • Spaces are part of tokens. They usually appear at the beginning of a token.
  • hello and hello are different tokens.
  • Numbers are grouped from left to right during tokenization.

Compression Ratio

  • $\text{compression ratio} = \text{num\_bytes} / \text{num\_tokens}$
  • Examples (GPT-4o):
    • Hello World
      • Hello(5) + space(1) + World(5) = 11 bytes
      • Tokens: 2
      • 5.5 bytes per token
    • 안녕 세상 (Korean)
      • 안(3) + 녕(3) + space(1) + 세(3) + 상(3) = 13 bytes
      • Tokens: 4
      • 3.25 bytes per token

Character-based Tokenization

  • Many characters rarely appear, making it highly inefficient.
  • Low compression ratio.

Byte-based Tokenization

  • 1 byte = 8 bits → $2^8 = 256$ possible values.
  • Vocab size: 256
  • Compression ratio: 1
  • Sequence length becomes very long. Since Attention is $O(n^2)$, computation cost increases significantly.

Word-based Tokenization

  • The number of words is extremely large, and many are rare.
  • Fixed vocab size cannot be determined.
  • e.g., ChatGPTized → <UNK>

Byte Pair Encoding (BPE)

  • Learns vocabulary by repeatedly merging frequently occurring sequences in text.
  • Starts with each byte as an initial token, then merges frequently occurring byte pairs across the entire text.
  • Frequent multi-character sequences become a single token.
  • Rare sequences are split into multiple tokens.
  • e.g., ChatGPTized → Chat, GPT, ized
  • e.g., ChatGPTized → Chat, G, PT, ized

Label Assignment

Sep 28, 2023
3 minute read
Korean

When training an Object Detection model, you first match the predicted candidate boxes with the Ground Truth (GT) boxes, and then compute the loss between each matched pair. This matching process is known as Label Assignment. Since improper matching can lead to incorrect learning, label assignment is a crucial step in the training pipeline. In this post, we will explore different label assignment techniques.

Table of Contents

  1. Fixed Label Assignment
  2. Dynamic Label Assignment

Fixed Label Assignment

  • Fixed Label Assignment allocates labels based on predefined thresholds.
  • IoU-based matching: If the Intersection over Union (IoU) between a predicted box and a GT box exceeds a certain threshold, the prediction is assigned to the GT.
  • Center-based matching: If the distance between the centers of the predicted box and GT box is within a threshold, the prediction is assigned to the GT.
  • Because fixed thresholds are used, there can be incorrect assignments in cases with overlapping GTs, potentially introducing confusion during training.


Source: Ge, Zheng, et al. “Ota: Optimal transport assignment for object detection.” Proceedings of the IEEE/CVF conference on computer vision and pattern recognition. 2021, Figure 1.

Dynamic Label Assignment

  • Bipartite Matching

    • Finds a globally optimal matching between predicted boxes and GTs so that each predicted box is assigned to at most one GT.
    • The image below illustrates bipartite matching in DETR, where each object query passes through the decoder and prediction heads. The Hungarian algorithm is then used to match queries to GTs or the “no object” class.
      Source: Carion, Nicolas, et al. “End-to-end object detection with transformers.” European conference on computer vision. Cham: Springer International Publishing, 2020, Figure 2.
    • Since each prediction is independently assigned to a GT, ambiguous cases are reduced and duplicate assignments are avoided.
    • However, the commonly used Hungarian algorithm has a high time complexity of O(n³), making it slow when there are many predictions or GTs.

  • OTA (Optimal Transport Assignment)

    • Defines the matching as an Optimal Transport (OT) problem, considering the matching cost between predicted boxes and GT boxes.
    • The cost is defined using cross-entropy loss for class prediction and IoU loss for location.
    • Once the cost matrix is constructed, the OT problem can be solved using the Sinkhorn-Knopp Iteration.
    • OTA algorithm illustration:

      Source: Ge, Zheng, et al. “Ota: Optimal transport assignment for object detection.” Proceedings of the IEEE/CVF conference on computer vision and pattern recognition. 2021, Algorithm 1.
    • Unlike bipartite matching, OTA allows one-to-many assignments.
    • One drawback is that solving the Sinkhorn algorithm can be time-consuming.

  • simOTA

    • Solving the OT problem with the Sinkhorn algorithm increases training time by approximately 25%.
    • simOTA simplifies the process by replacing the Sinkhorn algorithm with a top-K selection method.
    • First introduced in YOLOX.
    • The cost between GT $ g_i $ and prediction $ p_j $ is calculated as follows:
      • $ c_{ij} = L_{ij}^{cls} + \lambda L_{ij}^{reg} $
    • For each GT $ g_i $, predictions with the lowest top-K costs are assigned as positives.

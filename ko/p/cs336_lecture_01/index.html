CS336 Lecture 1: Tokenization

3월 18, 2026
1 분 정도
English

본 포스트에서는 Stanford CS336: Language Modeling from Scratch의 Lecture 1에서 다루는 Tokenization에 대해 정리합니다.

목차

  1. Tokenization

Tokenization

  • Tokenizer: encode와 decode를 수행하는 class입니다.
  • Vocabulary size: token 종류의 수입니다.

Tokenization 예시

tiktokenizer에서 직접 확인할 수 있습니다.

  • space도 token의 일부입니다. 보통 token 앞에 있습니다.
  • hello와 hello는 다른 token입니다.
  • 숫자는 왼쪽에서부터 묶어서 tokenize합니다.

Compression Ratio

  • $\text{compression ratio} = \text{num\_bytes} / \text{num\_tokens}$
  • 예시 (GPT-4o):
    • Hello World
      • Hello(5) + 공백(1) + World(5) = 11 bytes
      • 토큰 수: 2
      • 5.5 bytes per token
    • 안녕 세상
      • 안(3) + 녕(3) + 공백(1) + 세(3) + 상(3) = 13 bytes
      • 토큰 수: 4
      • 3.25 bytes per token

Character-based Tokenization

  • 많은 character들이 자주 등장하지 않는 character여서 매우 비효율적입니다.
  • Compression ratio가 낮습니다.

Byte-based Tokenization

  • 1바이트(byte)는 8비트(bit) → $2^8 = 256$가지 값을 표현 가능합니다.
  • Vocab size: 256
  • Compression ratio: 1
  • Sequence length가 길어집니다. Attention 연산이 $O(n^2)$이기 때문에 계산 비용이 커지는 단점이 있습니다.

Word-based Tokenization

  • Words의 수는 굉장히 크고, 그 중에는 자주 등장하지 않는 word도 많습니다.
  • Fixed vocab size가 정해지지 않습니다.
  • 예) ChatGPTized → <UNK>

Byte Pair Encoding (BPE)

  • 텍스트에서 자주 등장하는 sequence를 반복적으로 병합하여 vocabulary를 학습합니다.
  • 각 바이트를 초기 토큰으로 사용하고, 텍스트 전체에서 자주 등장하는 byte pair를 병합합니다.
  • 자주 나오는 여러 개 character의 sequence는 하나의 token이 됩니다.
  • Rare sequence는 여러 개의 token으로 분리됩니다.
  • 예) ChatGPTized → Chat, GPT, ized
  • 예) ChatGPTized → Chat, G, PT, ized
